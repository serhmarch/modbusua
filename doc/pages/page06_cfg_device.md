# Конфігурування об'єкта пристрою/ПЛК {#sec_cfg_device}

Об’єкт пристрою/ПЛК безпосередньо відповідає за роботу з відповідним пристроєм/ПЛК. 
Тут задаються всі необхідні параметри для налаштування протоколу Modbus та ін. параметрів для відповідного пристрою/ПЛК.

Конфігурація об'єкта пристрою/ПЛК задається у секції `[DEVICE]` файлу конфігурації `modbusua.conf`.

Список основних налаштувань конфігурації об'єкта пристрою/ПЛК:

| Назва                         | Опис                              
|:------------------------------|:----------------------------------
| `PortName`                    | Ім’я порту, до якого відноситься даний пристрій/ПЛК
| `Name`                        | Ім’я даного пристрою/ПЛК
| `EnableDevice`                | Доступність даного пристрою/ПЛК для ланцюга виконання (`1`-вкл, `0`-викл) 
| `ModbusUnit`                  | Modbus-адреса даного пристрою/ПЛК
| `RestoreTimeout`              | Тайм-аут відновлення зв’язку після помилки (виключення із ланцюга виконання) (мілісекунди)
| `MaxReadCoils`                | Максимальна кількість Coils для читання однією посилкою 
| `MaxWriteMultipleCoils`       | Максимальна кількість Coils для запису однією посилкою
| `MaxReadDiscreteInputs`       | Максимальна кількість DiscreteInputs для читання однією посилкою 
| `MaxReadInputRegisters`       | Максимальна кількість InputRegisters для читання однією посилкою 
| `MaxReadHoldingRegisters`     | Максимальна кількість HoldingRegisters для читання однією посилкою 
| `MaxWriteMultipleRegisters`   | Максимальна кількість HoldingRegisters для запису однією посилкою 
| `DefaultPeriod`               | Період опитування за замовчування для кожного Modbus Item
| `RequestTimeout`              | Таймаут запиту для кожного Modbus Item, після сплинення якого для кожного Modbus Item припиняє опитуватись
| `Log.flags`                   | Список прапорів, що задають рівень логування                                
| `Log.output`                  | Список типів журналювання для виводу (див. [Журналювання](#sec_journal_common).)                             
| `Log.file.format`             | Формат одного повідомлення для виводу у файл (див. [Журналювання](#sec_journal_common))                     
| `Log.file.timeformat`         | Формат часу одного повідомлення для виводу у файл (див. [Журналювання](#sec_journal_common))                
| `Log.file.path`               | Розташування файлу, у який будуть виводитись повідомлення                      
| `Log.file.maxcount`           | Максимальна кількість файлів для ротації журналу                              
| `Log.file.maxsize`            | Максимальний розмір файла ротації журналу (в байтах). Підтримуються суфікси: `KB`, `MB`, `GB` (наприклад, `5 MB`)                          

Список додаткових налаштувань конфігурації об'єкта пристрою/ПЛК:

| Назва                         | Опис                              
|:------------------------------|:----------------------------------
| `i:<item>=<itemreference>`    | Задає одиничний елемент даних з ім'ям `<item>` і посиланням `<itemreference>`
| `itemfile`                    | Ім’я файлу в якому окремо задається список елементів даних

Приклад налаштування одиничних елементів даних у секції `[DEVICE]` файлу конфігурації `modbusua.conf`:

```ini
[DEVICE]
...
i:Vlv2=VS.1
i:TestStatusBits ="VS.0.0,AS.0.0,TS.0.0,N.7,TS.0.33"
i:Test_Int16     ="400001 S"
i:Test_UInt16    ="400001 R"
i:Test_Int32     ="400001 I"
i:Test_Float     ="400001 F"
i:Test_Double    ="400001 LF"
i:Test_SwInt32   ="400001 IS"
i:Test_SwDouble  ="400001 LFS"
i:Test_ByteArray ="400001-400010 B"
...
```

Файл `itemfile` зберігає список елементів даних у форматі `<item>;<itemreference>`.
Приклад налаштування елементів даних у файлі `itemfile`:

```csv
...
"Analog001.Tune_Tmr_CtrlParamDown";"416605 L"
"Analog001.Tune_Tmr_CtrlParamDrop";"416603 L"
...
```

Формат `<itemreference>` заданий у [Modbus регістри](#sec_device_mb) і вкладених розділах.
