# Modbus регістри {#sec_device_mb}

Далі представлений формат конфігурації Modbus даних (item referenc):
`{0|1|3|4}xxxxx[-{0|1|3|4}yyyyy <arraysuffix>|<suffix>][,<period>][,msgid=<id>]`

`{0|1|3|4}<xxxxx>` - номер Modbus регістра, [000001-065536], [100001-165536], [300001-365536], [400001-465536]

`{0|1|3|4}<yyyyy>` - номер кінцевого Modbus регістра для випадку зчитування масиву регістрів, 
[000001-065536], [100001-165536], [300001-365536], [400001-465536]. 
При цьому зчитується масив починаючи з `{0|1|3|4}<xxxxx>` і закінчуючи `{0|1|3|4}<yyyyy>` включно;

`<arraysuffix>` - суфікс типу даних item reference для зчитування масиву регістрів
(обов’язковий параметр для масиву регістрів). Можливі значення:
- `B` – масив має тип `String`, який представляє собою послідовність байтів,
кожен з яких представлений двома символами шістнадцяткової системи числення (HEX)
(без пробілів або інших розділових знаків). 
Символи A-F повинні задаватись у верхньому регістрі.
Старші біти у байті  ідуть першими (зліва), молодші — останніми (справа).
При цьому молодший байт регістра іде першим (зліва), старший — другим (справа).
Наприклад для наступного знімку пам’яті ПЛК:
  | 400001 | 400002 |...| 400010 |
  |--------|--------|---|--------|
  | 0x0BCD | 0x80FE |...| 0xFC05 |

Результуючий `ItemReference` матиме вигляд:
  `400001-400010 B = "CD0BFE80...05FC"`

`<suffix>` - суфікс типу даних item reference (необов’язковий параметр). Суфікс описаний у розділі \ref sec_data_suffix. 
За замовчування (якщо суфікс не вказаний) тип даних приймається як `UInt16` для 3x та 4x регістрів і `Bool` для 0x та 1x.
Суфікс описаний у розділі \ref sec_data_suffix

`<period>` - період опитування даного `item` у мілісекундах.
Якщо період не вказаний, використовується `DefaultPeriod` із конфігурації пристрою.

`msgid=<id>` - ідентифікатор Modbus-посилки (необов’язковий параметр).\
В явному вигляді групує Modbus-регістри у Modbus-посилки. `<id>` може бути числом або словом.
