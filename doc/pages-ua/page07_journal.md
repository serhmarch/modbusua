# Журналювання роботи gateway {#sec_journal_common}

Під час виконання програми gateway до журналу можуть додаватись діагностичні повідомлення,
що відображають процес виконання програми.

Конфігурація журналу задається у секції `[SYSTEM]` наступними параметрами:

| Назва                     | Опис                                                                          
|:--------------------------|:------------------------------------------------------------------------------
| `Log.flags`               | Список прапорів, що задають рівень логування                                  
| `Log.output`              | Список типів журналювання для виводу (див. нижче)                             
| `Log.default.format`      | Формат одного повідомлення за замовчуванням (див. нижче)                      
| `Log.default.timeformat`  | Формат часу одного повідомлення за замовчуванням (див. нижче)                 
| `Log.system.format`       | Формат одного повідомлення для системного журналу (див. нижче)                
| `Log.system.timeformat`   | Формат часу одного повідомлення для системного журналу (див. нижче)           
| `Log.console.format`      | Формат одного повідомлення для виводу у консоль (див. нижче)                  
| `Log.console.timeformat`  | Формат часу одного повідомлення для виводу у консоль (див. нижче)             
| `Log.file.format`         | Формат одного повідомлення для виводу у файл (див. нижче)                     
| `Log.file.timeformat`     | Формат часу одного повідомлення для виводу у файл (див. нижче)                
| `Log.file.path`           | Розташування файлу, у який будуть виводитись повідомлення                     
| `Log.file.maxcount`       | Максимальна кількість файлів для ротації журналу                              
| `Log.file.maxsize`        | Максимальний розмір файла ротації журналу (в байтах). Підтримуються суфікси: `KB`, `MB`, `GB` (наприклад, `5 MB`)                          


Всі повідомлення розбиті по категоріям, щоб у подальшому мати можливість виводити ту інформація,
яка потрібна, наприклад, для відлагодження програми та ін.

Категорії задаються у файлі конфігурації параметром `Log.flags` для секції `[SYSTEM]`
у вигляді списку слів розділених символом `|`:

| Категорія         | Призначення                                                                          
|:------------------|:-------------------------------------------------------------------------------------
| `Error`           | Критичні помилки, що можуть призвести до зупинки або некоректної роботи програми     
| `Warning`         | Некритичні помилки, що не призводять до зупинки програми                             
| `Info`            | Загальні діагностичні повідомлення                                                   
| `Trace`           | Діагностичні повідомлення для деталізації роботи програми                            
| `TraceDetails`    | Деталізовані діагностичні повідомлення для деталізації роботи програми               
| `CtorDtor`        | Повідомлення про створення/видалення основних елементів програми                     
| `EntryExit`       | Повідомлення про початок/закінчення роботи важливих функцій програми                 
| `ThreadStartStop` | Повідомлення про старт/зупинку окремих програмних потоків системи                    
| `Connection`      | Повідомлення про встановлення/розірвання з’єднання із віддаленими об’єктами по мережі
| `Item`            | Повідомлення про створення/виконання окремих змінних ItemReference                   
| `Message`         | Повідомлення про створення/виконання окремих об’єктів-повідомлень (пакетів)          
| `Protocol`        | Специфічні повідомлення протоколу Modbus                  
| `Receive`         | Вхідні мережеві посилки протоколу Modbus                                             
| `Send`            | Вихідні мережеві посилки протоколу Modbus                                            

Для журналювання можуть використовуватись системний журнал, консоль, файл або їх комбінація. 
За замовчуванням для режиму сервісу(демону) використовується системний журнал,
в іншому випадку - консоль.

Вивід задається параметром `Log.output` у секції `[SYSTEM]` конфігураційного файлу, 
який може мати одне або декілька значень із списку (розділених пробілами):
* `system` - вивід у системний журнал
* `console` - вивід у консоль
* `file` - вивід у файл

Наприклад:
```ini
[SYSTEM]
Log.output="file console"
```

Параметри `Log.default.*` задають налаштування за замовчуванням для всіх способів виводу,
`Log.console.*` і `Log.file.*` відповідно для консолі і файлу (якщо вказані, перевизначають параметри за замовчуванням).

Параметри `Log.file.maxcount` і `Log.file.maxsize` призначенні для логування у файл
із використанням ротації кінцевих файлів.

Формат повідомлення може містити як звичайний текст для виводу так і спеціальні значення 
параметрів повідомлення, значення яких замінюються у момент формування повідомлення:

* `%%time` - мітка часу повідомлення, для якої також задається формат (`%%timeformat`)
* `%%text` - текст повідомлення
* `%%cat`  - текстове представлення категорії повідомлення
* `%%src`  - назва пристрою, порту, що видав подію журналу
             (або modbusua - для загальносистемних подій журналу)

Формат часу повідомлення (`%%timeformat`) має такі спеціальні символи:

* `%%Y` - рік (4 символи)
* `%%M` - місяць (2 символи `01`-`12`)
* `%%D` - день (2 символи `01`-`31`)
* `%%h` - година (2 символи `00`-`23`)
* `%%m` - хвилина (2 символи `00`-`59`)
* `%%s` - секунда (2 символи `00`-`59`)
* `%%f` - мілісекунда (3 символи `000`-`999`)

Приклад конфігруації для журналу:

```ini
[SYSTEM]

#...

Log.flags="Error|Warning|Info|TraceDetails"
Log.output="system file console"
Log.default.format="[%time] %cat %src. %text"
Log.default.timeformat="%Y-%M-%D %h:%m:%s.%f"
Log.concole.format="[%time] %src. %text"
Log.console.timeformat="%h:%m:%s.%f"
Log.file.format="%time;%cat;%src;%text"
Log.file.timeformat="%h:%m:%s.%f"
Log.file.path="log.csv"
Log.file.maxcount=10
Log.file.maxsize=1MB

#...

```

## Журналювання для окремого порту або пристрою {#sec_journal_common_sub}

Можна задати окремий вид журналу для відповідного порту або пристрою/ПЛК.
Список налаштувань із описаних вище, але доступних для порту/пристрою:
* `Log.flags`             
* `Log.output`            
* `Log.file.format`       
* `Log.file.timeformat`   
* `Log.file.path`         
* `Log.file.maxcount`     
* `Log.file.maxsize`

Відповідно для кожного порту/пристрою можна задавати свій список прапорів журналювання,
типів виводу (`system`, `console` та/або `file`) та параметрів файлу.

Параметри для виводу у консоль задаються  у секції `[SYSTEM]`.

Параметри виводу у файл можуть налаштовуватись індивідуально для кожного порту/пристрою
у відповідних секціях файлу конфігурації (`[PORT]` або `[DEVICE]`).

Якщо вказати однакове ім'я файлу виводу журналу для декількох портів/пристроїв,
то вивід буде відбуватись у один цей вказаний файл.

Якщо ніяких окремих виводів для порту/пристрою не вказано,
вивід буде здійснюватись у журнал, заданий у секції `[SYSTEM]`.

## Системний журнал ОС Windows {#sec_journal_windows}

Якщо параметр `Log.output` має значення `system` (або одне із значень списку є `system`),
то вивід буде здійснюватись у системний журнал ОС Windows.
Він знаходиться у журналі подій Windows (Event Viewer):
`Control Panel\System and Security\Administrative Tools\Event Viewer\Windows Logs\Application`.


## Системний журнал ОС Linux {#sec_journal_linux}

Для журналювання роботи gateway використовується системний журнал ОС Linux  `syslog/rsyslog`.
Для перегляду системного журналу диструбутивів Linux,
в яких використовується системний завантажувач `systemd`
можна використовувати програму `journalctl`.
Приблизний вигляд результатів роботи команди:
```console
$ journalctl -e -l -f -u modbusua --since "2025-08-06 13:00:00"
[...]
Aug 05 11:58:42 DESKTOP-TNLLA10 systemd[1]: Starting OPC UA - Modbus Gateway...
Aug 05 11:58:42 DESKTOP-TNLLA10 modbusua[121426]: [2025-08-05 11:58:42.656] modbusua. Loading config file: /etc/modbusua/modbusua.conf
Aug 05 11:58:42 DESKTOP-TNLLA10 systemd[1]: Started OPC UA - Modbus Gateway.
[...]
```

Основі параметри програми `journalctl`:
 Параметр       | Опис
----------------|----------------------------------------------------
`-a`            | Показати всі поля
`-e`            | Перехід у кінець журналу
`-l`            | Показати всі друковані поля журналу
`-n <num>`      | Показати останні `<num>`-записів у журналі
`-r`            | Огляд журналу із кінця (реверс)
`-f`            | Автоматичне оновлення журналу
`-u <unit>`     | Відображати записи лише для заданого unit (сервісу)
`--since <dt>`  | Відображати починаючи із заданої дати/часу
